# Умные бизнес-процессы

Приложение для Bitrix24 — каталог бизнес-процессов с генерацией документов и автоматическим запуском согласования.

## Возможности

- Каталог процессов в левом меню Bitrix24 (iframe)
- Универсальные формы заявок — поля настраиваются через панель администратора
- Генерация `.docx` документов из шаблонов с подстановкой переменных
- Автоматическая подтяжка подписи сотрудника из профиля Bitrix24
- Двухэтапное подписание: сотрудник → руководитель после согласования
- Запуск бизнес-процесса согласования при отправке заявки
- Панель администратора (доступна выбранным пользователям Bitrix24)
- Доступ только изнутри портала Bitrix24 — прямые запросы блокируются

## Быстрый старт

```bash
# 1. Скопировать переменные окружения
cp env.example .env

# 2. Заполнить .env (см. раздел Конфигурация)

# 3. Запустить
docker compose up -d

# 4. Открыть страницу установки в браузере с авторизованной сессией Bitrix24
# https://your-portal.bitrix24.ru/devops/edit/application/
# Вставить APP_URL/install как handler и нажать "Сохранить"
```

## Конфигурация (.env)

| Переменная | Описание |
|---|---|
| `CLIENT_ID` | Client ID OAuth-приложения Bitrix24 |
| `CLIENT_SECRET` | Client Secret OAuth-приложения |
| `APP_URL` | Публичный HTTPS URL этого сервиса |
| `BITRIX_WEBHOOK_URL` | Вебхук Bitrix24 для загрузки файлов |
| `DATABASE_URL` | Строка подключения PostgreSQL |
| `DB_PASSWORD` | Пароль PostgreSQL (для docker-compose) |
| `ADMIN_USER_IDS` | ID пользователей-администраторов через запятую |
| `SIGN_FIELD` | Код поля подписи в профиле сотрудника |

## Панель администратора

Доступна по адресу `/admin` внутри Bitrix24 (только для пользователей из `ADMIN_USER_IDS`).

Возможности:
- Добавить, отредактировать, включить/выключить процесс
- Настроить поля формы (типы: text, number, date, date_range, select, textarea, positions)
- Настроить подписи в шаблоне документа
- Указать ID шаблона `.docx` и папки на Диске Bitrix24

## Добавление нового процесса

1. Перейти в `/admin` → «Добавить процесс»
2. Заполнить ключ (латиница), название, иконку
3. Указать ID инфоблока, ID шаблона БП, ID файла-шаблона .docx, ID папки на Диске
4. Добавить маппинг полей инфоблока:
   ```json
   {"SUMMA": "PROPERTY_516", "NAZNACHENIE": "PROPERTY_518"}
   ```
5. Добавить поля формы — в каждом указать имя переменной (должно совпадать с `${VAR}` в шаблоне `.docx`)
6. Настроить подписи (SIGN_EMPLOYEE, SIGN_APPROVER)
7. Сохранить — процесс появится в меню у всех пользователей

## Шаблоны документов (.docx)

Плейсхолдеры в тексте: `${VARIABLE_NAME}`

Плейсхолдеры подписей: изображения-заглушки с Alt Text = имени placeholder
(например `SIGN_EMPLOYEE`, `SIGN_APPROVER`)

## Структура проекта

```
app/
├── main.py            # FastAPI app + middleware + роутеры
├── config.py          # Настройки (pydantic-settings)
├── database.py        # SQLAlchemy async engine
├── seed.py            # Начальные данные (процесс cash)
├── models/            # ORM-модели: Process, FormField, DocumentSignature
├── routers/           # Маршруты: install, app, api, submit, activity, admin
├── services/          # bitrix.py (httpx), document.py (python-docx)
├── middleware/        # Проверка источника Bitrix24 + доступ к /admin
└── templates/         # HTML-шаблоны: menu, form, admin/*
migrations/            # Alembic миграции
```

## API

| Метод | Путь | Описание |
|---|---|---|
| GET/POST | `/install` | Установка приложения Bitrix24 |
| GET/POST | `/app` | Каталог процессов / форма заявки |
| GET | `/api/process/{key}` | Конфиг формы в JSON |
| POST | `/submit` | Отправка заявки |
| POST | `/activity-handler` | Генерация документа из активити БП |
| GET/POST | `/admin` | Панель администратора |
| GET | `/health` | Проверка работоспособности |
